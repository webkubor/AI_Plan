## AI 工程化术语表｜由浅入深

>
目标：从基础概念 → 工具能力 → 复用能力 → 智能执行 → 工程化编排 → 标准化接入 → 多智能体协作 → 应用层方法 → 未来趋势。
>
0. 一句话总览（先给整体框架）
==============================
大模型从“对话框”进化为“生产力系统”的关键路径：
LLM（大脑） → Tool/Function（执行能力） → Skill（可复用模块） → Agent（自主执行者）
→ Workflow（确定性流水线） → MCP（工具接入标准） → Multi-Agent（团队协作）
→ RAG/Few-shot/Structured Reasoning（可靠性方法） → Edge AI & AI-Native UI（趋势）
1. 基础概念层（模型是什么）
==============================



1) LLM（Large Language Model，大语言模型）
- 定义：具备语言理解与生成能力的模型，可生成文本、代码、推理过程等。
- 对应工具/应用：GPT-4、Claude 3、Llama 3、Gemini
2) Code LLM（代码大模型）
- 定义：专为编程优化的 LLM，擅长代码补全、重构、生成测试等任务。
- 对应工具/应用：CodeLlama、StarCoder、Codex、Cursor
3) Zero-shot Learning（零样本学习）
- 定义：无需示例，模型直接完成任务。
- 对应工具/应用：Gemini Ultra、Llama 3 70B、Claude 3、GPT-4
4) Few-shot Learning（少样本学习）
- 定义：提供少量示例，快速对齐输出结构、风格、格式等。
- 对应工具/应用：GPT-4 Turbo、Claude 3 Opus、Gemini Pro
5) Chain of Thought（CoT）/ Structured Reasoning（结构化推理）
- 定义：让模型分步骤推理；工程上更推荐将其表达为固定的推理流程（如 Plan → Execute → Verify）。
- 对应工具/应用：ReAct、Plan&Execute、Reflexion（推理框架）
2. 外部执行层（模型如何“动手”）
==============================
6) Tool（工具 / Action）
- 定义：模型可调用的外部能力单元（API、脚本、数据库查询、执行命令等）。
- 对应工具/应用：API、SDK、CLI、数据库、内部服务
7) Tool Calling（工具调用）
- 定义：模型根据需要主动调用外部工具来获取信息或执行动作。
- 对应工具/应用：GPT Function Calling、Claude Tool Use、Ollama Function Calling
8) Function Calling（函数调用）
- 定义：Tool Calling 的一种实现方式：模型调用预定义函数，并输出结构化参数（如 JSON）。
- 对应工具/应用：GPT-4 Function Calling、Claude Tool Use、Ollama Function Calling



3. 能力复用层（把经验变成模块）
==============================



9) Skill（可复用技能单元）
- 定义：一段可复用的方法/步骤（通常封装为 Tool），供 Agent 调用。
- 特点：可控、可测试、可复用、可组合。
- 对应工具/应用：LangChain Tools、Zapier Actions、Tool Registry
4. 自主执行层（从工具到“执行者”）
==============================

、

10) Agent（目标 + 规划 + 决策 + 多步执行）
- 定义：具备目标意识与决策能力的系统，可多步规划并调用 Skills/Tools 完成复杂任务。
- 特点：强自主性、适合非确定性问题、可迭代修正。
- 对应工具/应用：AutoGPT、LangChain Agent、Microsoft Copilot Studio、AutoGen、Swarm
5. 工程化编排层（让执行“可控”）
==============================



11) Workflow（确定性流程 / 流水线）
- 定义：固定步骤的执行流程，通过编排系统约束顺序、条件、重试与回滚。
- 核心价值：
  - 确定性（同输入同输出）
  - 可观测性（日志、审计、追踪）
  - 可回滚性（失败回退、补偿机制）
  - 可复现性（可复跑、可验证）
- 对应工具/应用：Airflow、Prefect、Dagster、Temporal、LangFlow、MuleSoft



6. 工具接入标准层（解决“重复对接”）
==============================



12) MCP（Model Context Protocol，模型上下文协议）
- 定义：面向 LLM 工具生态的标准化接入协议，使工具对接像 USB 一样即插即用。
- 解决的问题：不同工具/系统反复对接的重复劳动；工具发现、复用、权限与上下文管理。
- 对应工具/应用：
  - Anthropic MCP（协议/生态）
  - mcp-server-*（各类工具服务实现）
  - Claude Desktop / Cursor / Continue（MCP Client）

7. 多智能体协作层（从“一个人”到“一个团队”）
==============================



13) Multi-Agent System（多智能体系统）
- 定义：多个 Agent 分工协作完成复杂任务，通过角色划分、通信与协调实现整体目标。
- 对应工具/应用：ChatGPT Team、AutoGen、Swarm

8. 应用方法层（让输出更可靠）
==============================



14) RAG（Retrieval-Augmented Generation，检索增强生成）
- 定义：生成前先检索外部知识库，再基于检索结果生成内容。
- 关键价值：减少幻觉、补足私有知识、提升可信度。
- 对应工具/应用：LangChain、LlamaIndex、Pinecone、Weaviate、Milvus、Chroma
15) Embedding（嵌入）
- 定义：把文本/代码转换为高维向量表示，用于相似性检索。
- 对应工具/应用：OpenAI Embeddings、Sentence-BERT、Hugging Face Embeddings

16) Vector Database（向量数据库）
- 定义：存储向量并提供相似度检索能力，常作为 RAG 的底座。
- 对应工具/应用：Pinecone、Milvus、Chroma、Weaviate

17) Orchestration（编排）
- 定义：协调 LLM、Agent、Tools、Workflow 之间的调用与协作，形成系统级执行能力。
- 对应工具/应用：LangChain、Airflow、Prefect、Dagster、Temporal
18) API Gateway for AI（AI 网关）
- 定义：统一管理多模型/多工具调用的中间层，用于权限控制、流量管理、成本统计等。
- 对应工具/应用：Kong AI、AWS API Gateway、FastAPI

9. 未来趋势层（下一步演进方向）
==============================



19) Edge AI（边缘 AI）
- 定义：模型运行在本地设备（浏览器/手机/PC），强调隐私与低延迟。
- 对应工具/应用：WebGPU、WebAssembly、Transformers.js、ONNX Runtime Web、TensorFlow.js
20) AI-Native UI / UX（AI 原生交互）
- 定义：以自然语言意图驱动的交互范式，UI 动态适配任务与上下文。
- 对应工具/应用：ChatGPT Apps、Gemini Apps、Perplexity、各类 Agent UI

21) Generative UI（生成式 UI）
- 定义：界面由模型实时生成与调整，而非固定菜单与静态页面。
- 对应工具/应用：Vercel v0、Figma AI、Uizard、Google Stitch



22) Responsible AI（负责任 AI）
- 定义：关注公平性、透明性、安全性、隐私与版权风险控制。
- 对应工具/应用：IBM AI Fairness 360、Google What-If Tool、OpenAI Moderation API
